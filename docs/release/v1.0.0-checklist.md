## 4. Pre-release blockers (must resolve before `v1.0.0` tag)

Ordered by severity:

1. **RED — Overlap-disclosure cleanup in ux + growth packs.** Both packs ship with zero `Complementa matilha-*` / `No direct sibling` statements in skill bodies. The discipline was formalized in Wave 5b and is consistently applied in harness/sysdesign/software-eng packs. Shipping v1.0.0 with this gap leaves the consistency claim vulnerable. Estimated work: 2-3h per pack (identify real sibling overlaps first, then patch skills). Owner: pack maintainer (Danilo).

2. **RED — README consistency across older packs.** ux, growth, and harness READMEs lack (a) explicit user-scope install recommendation and (b) a "Relationship to matilha core" / Companion Integration section naming `matilha-compose`. Sysdesign and software-eng set the template. Estimated work: 30min per pack.

3. **RED — `matilha` CLI repo has no tag reflecting current main.** Latest tag is `wave-3b-complete`; since then Wave 4a through Wave 5f validators have shipped (pack validator suites bring the test count from 343 → 1211). Tag alignment required before any "matilha v1.0.0" claim. Owner: Danilo.

4. **RED — No `v1.0.0` CHANGELOG entry in `matilha-skills` core** (nor in CLI). The tag commit needs a changelog section that (a) declares v1.0.0, (b) summarizes the ecosystem state, (c) explicitly notes which pieces bump and which stay (see version-bump-strategy.md).

5. **YELLOW — marketplace.json `owner.email` inconsistency.** Uses `danilod2.sousa@gmail.com` (public persona) rather than the user-profile email. Intentional? Confirm before tag — the marketplace submission will surface this address.

6. **YELLOW — CLI npm-publishing decision.** The task says "do not bump versions", but v1.0.0 announcement implies users can install via npm. If CLI stays unpublished, announcement must route users to plugin install only. Decision needed; not a tag-blocker.

7. **YELLOW — `matilha-skills` tag hygiene.** Current main at `34c5125` is post-`wave-5d1-storytelling`. Either advance the wave tag to current main before cutting v1.0.0, or cut v1.0.0 directly from current main (recommended).

## 6. Verification run-order (recommended)

Before cutting the v1.0.0 tag, execute in order:

1. **Fix blockers #1, #2, #4** in separate commits on each affected repo (do NOT touch outside docs/release/ in this session — these are follow-up tasks).
2. Re-run `npm test` in `matilha` CLI → confirm `1211 / 1211` still green.
3. Manually run `hooks/session-start` → confirm JSON parses + exit 0.
4. In a fresh Claude Code workspace with matilha installed at user scope, run the Wave 5d governança prompt → confirm sigil emits + pack detection naming matches installed packs.
5. Cut tags in this order: (a) each companion pack, (b) `matilha-skills` core, (c) `matilha` CLI.
6. Update `matilha-skills/README.md` v1.0.0 badge + CHANGELOG simultaneously with the tag commit.
